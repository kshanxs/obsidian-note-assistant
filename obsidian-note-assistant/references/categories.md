# Category Selection Guide

Use this guide to determine which folder a draft note belongs in.

## Categories

### Ideas/Coding
For: software projects, app concepts, coding experiments, technical architecture ideas, AI tools, scripts, developer workflows.

**Trigger phrases:** "I want to build", "app idea", "feature idea", "API", "script", "algorithm", "database", "deployment", "CLI", "automation"

**Examples:**
- A concept for a habit-tracking mobile app
- An idea for a CLI tool that renames files by AI
- Notes on a new backend architecture

---

### Ideas/Book
For: book ideas, writing projects, story concepts, characters, plot outlines, worldbuilding notes, chapter drafts or summaries.

**Trigger phrases:** "chapter", "character", "story", "novel", "plot", "worldbuilding", "book idea", "narrative"

**Examples:**
- A sci-fi story concept set in 2087 Mumbai
- A character sketch for a protagonist
- Rough chapter outline

---

### Ideas/Trip
For: travel plans, destination ideas, itineraries, packing lists, budget notes, travel reflections.

**Trigger phrases:** "trip to", "travel", "visit", "itinerary", "flight", "hotel", "passport", "explore", "destination"

**Examples:**
- Weekend trip to Coorg
- Europe backpacking plan for summer
- Notes on what to pack for a mountain trek

---

### Daily
For: daily logs, journal entries, morning/evening reflections, mood tracking, gratitude notes, what happened today.

**Trigger phrases:** "today", "yesterday", "this morning", "journal", "reflection", "how I'm feeling", "daily"

**Examples:**
- Today's standup notes
- Evening reflection on a tough conversation
- Morning intention-setting note

---

### Knowledge
For: learning notes, book summaries, article takeaways, concept explanations, research notes, "TIL" moments, mental models.

**Trigger phrases:** "I learned", "reading about", "summary of", "concept", "notes on", "mental model", "book summary", "research"

**Examples:**
- Summary of "Atomic Habits" chapter 3
- Notes on transformer architecture
- A mental model for decision-making under uncertainty

---

### Projects
For: active projects with tasks, timelines, and progress tracking. Different from Ideas — these have defined scope and ongoing work.

**Trigger phrases:** "project", "milestone", "deadline", "in progress", "sprint", "task list", "launch", "team", "status update"

**Examples:**
- A freelance client project with deliverables
- A personal project (building a website) with task tracking
- Launch plan for a side project

---

## When two categories seem equally valid

Prefer the more **specific** category. For example:
- A coding-related book summary → `Knowledge` (not `Ideas/Coding`)
- A trip that's also a work project → `Projects` (if it has tasks/deliverables)
- A vague "idea" with no detail → `Ideas/Coding` or `Ideas/Book` based on domain

When genuinely ambiguous, pick the best fit and tell the user why.
